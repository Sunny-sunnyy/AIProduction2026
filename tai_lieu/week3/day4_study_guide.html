Core Theory - Lý thuyết cốt lõi

Key concepts - Khái niệm chính

  • Stateful vs Stateless Resources - Tài nguyên có trạng thái và không trạng thái: Tài nguyên stateful (S3 Vector bucket, DB) lưu trữ dữ liệu bền vững cần được bảo vệ. Tài nguyên stateless (Lambda, API Gateway) có thể phá hủy và tái tạo tự động bằng mã nguồn hạ tầng.
  • Cosine similarity - Độ tương đồng Cosine: Phép đo hướng của hai vector trong không gian đa chiều, đánh giá độ gần gũi ngữ nghĩa từ -1 đến 1 (càng gần 1 càng đồng nghĩa).
  • Lambda Deployment Package - Gói triển khai Lambda: File nén `.zip` chứa mã nguồn và toàn bộ dependencies giúp Lambda thực thi độc lập trên môi trường đám mây AWS.
  • Usage Plan - Kế hoạch sử dụng: Cơ chế kiểm soát API Gateway, liên kết API Key với các giới hạn về lưu lượng gọi (quota) và tốc độ gọi (throttling/burst).
  • Throttling and Burst - Điều tiết và Bùng nổ: Cơ chế chống quá tải. Rate limit (số yêu cầu trung bình mỗi giây) và Burst limit (số yêu cầu xử lý ngay lập tức tại một thời điểm rất ngắn).

S3 Vectors vs OpenSearch Serverless

Đặc trưng S3 Vectors OpenSearch Serverless
Chi phí (Cost) Rất rẻ (~$20-30/tháng) nhờ co giãn về 0. Rất đắt (~$200-300/tháng) do phí duy trì cụm tối thiểu.
Độ phức tạp Rất đơn giản, tích hợp trực tiếp trên S3 Console. Cao, yêu cầu cấu hình mạng VPC, bảo mật và quyền truy cập phức tạp.
Độ trễ Trung bình/Thấp (phụ thuộc S3 I/O). Cực kỳ thấp (tối ưu hóa bộ nhớ đệm).
Tính năng Hỗ trợ thuật toán tìm kiếm khoảng cách cơ bản. Hỗ trợ tìm kiếm kết hợp (hybrid search), bộ lọc văn bản chuyên sâu.
Deep explanation - Giải thuật tìm kiếm lân cận HNSW trên S3 Vectors
Kiến thức mở rộng

Hierarchical Navigable Small World (HNSW): Là giải thuật tìm kiếm lân cận gần đúng (Approximate Nearest Neighbor - ANN) được tích hợp trực tiếp dưới tầng lưu trữ vật lý của S3 Vectors.

Cách thức hoạt động:

  • HNSW xây dựng một cấu trúc đồ thị phân cấp nhiều lớp (tương tự skip-list). Lớp trên cùng chứa các nút thưa thớt giúp định tuyến nhanh đến khu vực gần đúng của vector truy vấn.
  • Càng xuống lớp dưới, mật độ nút càng dày đặc để tìm kiếm chi tiết. Điều này cho phép S3 Vectors thực hiện tìm kiếm ngữ nghĩa trong thời gian mili-giây trên hàng triệu vector mà không cần quét tuyến tính qua tất cả các đối tượng S3, khắc phục nhược điểm về tốc độ đọc/ghi của S3 thô.

Bước 4: Tra cứu ngữ nghĩa và tính toán điểm tương đồng (Semantic Search)

  3. Script gọi API s3_vectors.query_vectors truyền query vector và tham số topK=3, returnDistance=True.
  4. S3 Vectors thực hiện thuật toán tìm kiếm lân cận HNSW trên S3 bucket, trả về danh sách các vector có khoảng cách ngắn nhất kèm theo giá trị distance (khoảng cách Cosine).

Techniques - Kỹ thuật sử dụng

API Rate Limiting - Giới hạn tần suất API

Purpose: Ngăn chặn tấn công DDoS và tránh các lỗi vòng lặp vô hạn ở client làm tăng chi phí AWS khổng lồ từ Lambda và SageMaker.

When to use: Bắt buộc áp dụng cho mọi API Gateway REST API public hướng ra ngoài internet trong các hệ thống sản xuất.

Trade-off: Làm tăng độ phức tạp trong cấu hình hạ tầng và yêu cầu quản lý phân phối API Key cho các client.

Stateful Resource Isolation - Cô lập tài nguyên có trạng thái

Purpose: Đảm bảo an toàn dữ liệu, tránh việc vô tình xóa mất cơ sở dữ liệu vector hoặc tệp dữ liệu gốc khi cập nhật hạ tầng stateless.

When to use: Áp dụng cho các cơ sở dữ liệu, S3 Vector buckets trong môi trường học tập/thử nghiệm có tần suất hủy hạ tầng cao bằng Terraform.

Trade-off: Đòi hỏi nhà phát triển phải thực hiện tạo thủ công bằng tay hoặc duy trì một kịch bản Terraform quản lý trạng thái riêng biệt.

API Gateway Auto-Redeployment Trigger

Purpose: Tự động kích hoạt redeployment stage prod của API Gateway khi có bất kỳ thay đổi cấu hình nào trong file code Terraform.

When to use: Khi làm việc với tài nguyên aws_api_gateway_deployment của Terraform để tránh tình trạng hạ tầng cập nhật nhưng API thực tế vẫn chạy phiên bản cũ.

Trade-off: Sử dụng hàm băm sha1(jsonencode(...)) làm tăng một chút thời gian kiểm tra thay đổi của Terraform engine.

# Hàm gọi SageMaker Endpoint tạo vector embedding 384 chiều
def get_embedding(text):
    response = sagemaker_runtime.invoke_endpoint(
        EndpointName=SAGEMAKER_ENDPOINT,     # Tên endpoint cấu hình từ biến môi trường
        ContentType='application/json',
        Body=json.dumps({'inputs': text})    # Truyền chuỗi văn bản cần nhúng
    )
    
    result = json.loads(response['Body'].read().decode())
    
    # Do container Hugging Face trả về mảng lồng 3 lớp [[[embedding]]] cho tác vụ feature-extraction,
    # chúng ta thực hiện bóc tách mảng về mảng 1 chiều chứa 384 phần tử số thực.
    if isinstance(result, list) and len(result) > 0:
        if isinstance(result[0], list) and len(result[0]) > 0:
            if isinstance(result[0][0], list):
                return result[0][0]  # Trích xuất từ [[[embedding]]]
            return result[0]  # Trích xuất từ [[embedding]]
    return result  # Trả về nguyên bản nếu cấu trúc không bị lồng

# Lambda Handler chính tiếp nhận yêu cầu từ API Gateway
def lambda_handler(event, context):
    try:
        # Hỗ trợ phân tích cả chuỗi JSON string thô gửi qua AWS PROXY Integration
        if isinstance(event.get('body'), str):
            body = json.loads(event['body'])
        else:
            body = event.get('body', {})
        
        text = body.get('text')
        metadata = body.get('metadata', {})
        
        if not text:
            return {
                'statusCode': 400,
                'body': json.dumps({'error': 'Missing required field: text'})
            }
        
        # 1. Gọi SageMaker sinh vector nhúng
        embedding = get_embedding(text)
        
        # 2. Sinh mã UUID v4 làm khóa duy nhất cho vector
        vector_id = str(uuid.uuid4())
        
        # 3. Ghi vector nhúng và metadata gốc vào S3 Vectors bucket
        s3_vectors.put_vectors(
            vectorBucketName=VECTOR_BUCKET,
            indexName=INDEX_NAME,            # financial-research
            vectors=[{
                "key": vector_id,
                "data": {"float32": embedding}, # Định dạng float32 bắt buộc của S3 Vectors
                "metadata": {
                    "text": text,
                    "timestamp": datetime.datetime.utcnow().isoformat(),
                    **metadata                # Đính kèm metadata tùy chọn (ticker, company_name,...)
                }
            }]
        )
        
        return {
            'statusCode': 200,
            'body': json.dumps({
                'message': 'Document indexed successfully',
                'document_id': vector_id
            })
        }
    except Exception as e:
        print(f"Error: {str(e)}")
        return {
            'statusCode': 500,
            'body': json.dumps({'error': str(e)})
        }

[package.py](file:///G:/AIProduction_t6_2026/production/week3/alex/backend/ingest/package.py) - Script đóng gói Lambda tự động

Script Python thực hiện cài đặt các thư viện phụ thuộc từ môi trường ảo .venv của uv và nén mã nguồn thành tệp zip.

def create_deployment_package():
    current_dir = Path(__file__).parent
    build_dir = current_dir / 'build'
    package_dir = build_dir / 'package'
    zip_path = current_dir / 'lambda_function.zip'
    venv_site_packages = current_dir / '.venv' / 'lib'
    
    # 1. Dọn dẹp các thư mục build cũ để tránh nén trùng lặp dữ liệu thừa
    if build_dir.exists():
        shutil.rmtree(build_dir)
    if zip_path.exists():
        os.remove(zip_path)
    
    package_dir.mkdir(parents=True, exist_ok=True)
    
    # 2. Định vị thư mục site-packages của môi trường ảo (Cross-platform)
    site_packages = None
    for path in venv_site_packages.rglob('site-packages'):
        site_packages = path
        break
        
    # 3. Sao chép toàn bộ dependencies vào thư mục tạm thời
    for item in site_packages.iterdir():
        if item.name.endswith('.dist-info') or item.name == '__pycache__':
            continue
        if item.is_dir():
            shutil.copytree(item, package_dir / item.name, dirs_exist_ok=True)
        else:
            shutil.copy2(item, package_dir)
            
    # 4. Sao chép file mã nguồn Lambda vào thư mục tạm thời
    shutil.copy(current_dir / 'ingest_s3vectors.py', package_dir)
    
    # 5. Tạo tệp tin nén lambda_function.zip để sẵn sàng deploy qua Terraform
    with zipfile.ZipFile(zip_path, 'w', zipfile.ZIP_DEFLATED) as zipf:
        for root, dirs, files in os.walk(package_dir):
            dirs[:] = [d for d in dirs if d != '__pycache__']
            for file in files:
                file_path = Path(root) / file

                zipf.write(file_path, arcname)
                
    shutil.rmtree(build_dir) # Dọn dẹp thư mục build tạm thời

Đoạn mã nguồn trong kịch bản kiểm thử tính toán điểm số tương đồng ngữ nghĩa từ khoảng cách Cosine trả về từ S3 Vectors.

def search_vectors(query_text, k=5):
    # 1. Gọi SageMaker chuyển câu hỏi thành vector đặc trưng
    query_embedding = get_embedding(query_text)
    
    # 2. Truy vấn trực tiếp cơ sở dữ liệu S3 Vectors
    response = s3_vectors.query_vectors(
        vectorBucketName=VECTOR_BUCKET,
        indexName=INDEX_NAME,
        queryVector={"float32": query_embedding}, # Truyền vector truy vấn
        topK=k,
        returnDistance=True,                     # Yêu cầu trả về khoảng cách Cosine
        returnMetadata=True                      # Yêu cầu trả về metadata văn bản gốc
    )
    
    vectors = response.get('vectors', [])
    print(f"Found {len(vectors)} results:\n")
    
    # 3. Duyệt kết quả và tính toán Similarity Score
    for vector in vectors:
        metadata = vector.get('metadata', {})
        distance = vector.get('distance', 0)
        
        # Công thức đổi từ khoảng cách Cosine sang điểm số tương đồng trực quan [0, 1]
        similarity_score = 1 - distance
        print(f"Score: {similarity_score:.3f}")
        if metadata.get('company_name'):
            print(f"Company: {metadata['company_name']} ({metadata.get('ticker', 'N/A')})")
        print(f"Text: {metadata.get('text', '')[:200]}...")
        print()

Options / Trade-offs - Bản đồ lựa chọn

Lựa chọn kiến trúc Phù hợp khi (Best for) Đánh đổi (Trade-off) Rủi ro (Risk)
Lựa chọn 1: Lưu trữ vector bucket trực tiếp qua Terraform
Khai báo tài nguyên S3 Vector Bucket trong tệp code main.tf của Terraform.		 Khi hệ thống chạy trên pipeline CI/CD ổn định, không có nhu cầu gỡ bỏ hoặc tái tạo hạ tầng thường xuyên. Toàn bộ dữ liệu vector đã được nạp sẽ bị xóa sạch hoàn toàn mỗi khi nhà phát triển chạy lệnh terraform destroy. Mất mát dữ liệu tri thức của hệ thống RAG khi dọn dẹp hạ tầng stateless.
Lựa chọn 2: Cô lập tài nguyên trạng thái (Lựa chọn của dự án)
Khởi tạo Vector Bucket thủ công trên AWS Console độc lập.		 (Recommended) Môi trường phát triển, học tập thực hành có nhu cầu hủy/tạo liên tục phần hạ tầng API Gateway và Lambda tính toán. Đòi hỏi nhà phát triển phải thực hiện tạo thủ công bằng tay và khai báo chính xác tên bucket vào tệp cấu hình .env. Gây sai sót cấu hình biến môi trường cục bộ nếu gõ sai tên bucket ảo.
Lựa chọn 3: Public Endpoint không yêu cầu API Key
Tắt thuộc tính xác thực API Key trên method POST của API Gateway.		 Chỉ dùng cho các buổi demo ngắn hạn hoặc môi trường mạng nội bộ hoàn toàn cô lập, không có kết nối internet. Bất kỳ ai dò được API URL đều có thể spam tài nguyên backend. Cạn kiệt tài nguyên hệ thống và phát sinh chi phí khổng lồ từ SageMaker endpoint.
Section 9

Pitfalls - Lỗi / bẫy thường gặp

Failure modes - Trường hợp dễ lỗi

  • Lỗi ghi dữ liệu vào S3 Vectors do sai cấu trúc mảng: SageMaker trả về mảng lồng 3 lớp [[[embedding]]]. Nếu truyền thẳng mảng này vào API put_vectors, S3 Vectors sẽ từ chối và báo lỗi API 500.
  • Lỗi gọi API Gateway trả về statusCode 403 hoặc 500: Sau khi deploy thành công, gọi endpoint từ curl nhận lỗi Forbidden hoặc Internal Server Error.
  • Phát sinh chi phí lớn ngoài ý muốn trên tài khoản AWS: Học viên để các cụm OpenSearch cũ hoặc các RDS Instance chạy liên tục không tắt.

Fix / prevention - Cách khắc phục

  • Khắc phục sai định dạng: Sử dụng khối lệnh kiểm tra kiểu dữ liệu isinstance(..., list) trong hàm get_embedding để bóc tách mảng lồng về mảng 1 chiều chứa 384 số thực trước khi ghi.
  • Khắc phục API lỗi: Xác minh tài nguyên aws_lambda_permission.api_gateway được khai báo đúng để cấp quyền cho API Gateway invoke Lambda, và kiểm tra API Key đã được gán vào Usage Plan stage prod.
  • Khắc phục chi phí ngầm: Sử dụng S3 Vectors co giãn về 0. Thiết lập AWS Budgets và hủy (destroy) các tài nguyên dọn dẹp RDS khi kết thúc buổi thực hành.
Section 10

Flashcards / Q&A - Thẻ ôn tập

Q: Tại sao phải tách S3 Vector Bucket ra khỏi quản lý của Terraform trong bài học này?

A: Để cô lập tài nguyên có trạng thái (stateful). Khi chạy lệnh `terraform destroy` để hủy hạ tầng tính toán stateless (Lambda, API Gateway), dữ liệu vector tri thức được nạp thủ công sẽ không bị xóa mất.

Q: Công thức tính Similarity Score trong bài tìm kiếm ngữ nghĩa là gì và tại sao cần công thức này?

A: Công thức là `Similarity = 1 - Cosine Distance`. Cần công thức này vì khoảng cách Cosine từ S3 Vectors trả về dải trị từ [0, 2], chuyển đổi giúp đưa điểm số tương đồng về dải trị trực quan từ [0, 1] (điểm càng cao nghĩa càng sát).

Q: Tác dụng của Usage Plan trong API Gateway là gì?

A: Usage Plan giúp giới hạn số lượng cuộc gọi tối đa của client theo tháng (quota) và tốc độ gọi tối đa mỗi giây (throttling/burst), ngăn chặn client spam hoặc vô tình lặp cuộc gọi gây tốn chi phí.

Q: Kỹ thuật sha1 băm cấu hình trong aws_api_gateway_deployment dùng để làm gì?

A: Giúp Terraform tự động phát hiện mọi thay đổi cấu hình nhỏ nhất trong API Gateway (route, method, integration) để thực hiện redeployment lại stage ngay lập tức, tránh hiện tượng cấu hình lệch pha với thực tế trên AWS.

Q: Tại sao tìm kiếm ngữ nghĩa (semantic search) lại vượt trội hơn tìm kiếm từ khóa (keyword search)?

A: Vì tìm kiếm ngữ nghĩa so khớp dựa trên ý nghĩa ngữ cảnh và khái niệm, cho phép tìm thấy tài liệu liên quan ngay cả khi từ ngữ hỏi và văn bản gốc hoàn toàn khác nhau (ví dụ: "electric vehicles" tìm thấy tài liệu chứa "TSLA").
