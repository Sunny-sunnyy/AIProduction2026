Mục 4

Core Theory - Lý thuyết cốt lõi

Docx LLM Slide

Khái niệm Kiến trúc

  • Frontend - giao diện người dùng: Mã nguồn chạy trên browser (trình duyệt) của client, đảm nhận thiết kế cấu trúc (HTML), giao diện (CSS) và tương tác (JS/TS).
  • Backend - máy chủ dịch vụ: Logic chạy trên server, quản lý cơ sở dữ liệu, gọi API OpenAI và bảo mật secrets (khóa bí mật).
  • API contract - hợp đồng API: Định dạng dữ liệu thống nhất (thường là JSON) giúp frontend và backend giao tiếp không bị lỗi.
  • Single Page Application (SPA) - ứng dụng đơn trang: Chỉ tải trang một lần, quá trình chuyển trang tiếp theo chỉ vẽ lại giao diện qua dữ liệu từ API.

React & Next.js Cơ bản

  • Document Object Model (DOM) - mô hình đối tượng tài liệu: Cấu trúc cây biểu diễn HTML của trình duyệt.
  • Props - thuộc tính component: Dữ liệu cấu hình truyền từ cha xuống con, có tính chất read-only (chỉ đọc) ở component con.
  • State - trạng thái dữ liệu: Dữ liệu nội bộ của component, khi thay đổi sẽ tự động kích hoạt re-render giao diện.
  • Transpiling - biên dịch mã: Chuyển đổi mã nguồn bậc cao (như TypeScript) sang một ngôn ngữ bậc cao khác (như JavaScript) để trình duyệt hiểu được.
Giải thích sâu về Virtual DOM trong React

React không can thiệp trực tiếp vào DOM thật của trình duyệt vì việc đó tốn rất nhiều tài nguyên tính toán. Thay vào đó, React sử dụng một bản sao nhẹ nằm trên RAM gọi là Virtual DOM - DOM ảo. Khi state (trạng thái) của component thay đổi:

  1. React tự động dựng một cây DOM ảo mới.
  2. Chạy thuật toán so sánh (diffing algorithm) để tìm ra sự khác biệt nhỏ nhất giữa cây DOM ảo mới và cây DOM ảo cũ.
  3. Chỉ cập nhật những thay đổi thực sự đó (gọi là reconciliation) lên DOM thật của trình duyệt, giúp giao diện phản hồi cực nhanh.

Quy trình hoạt động chi tiết đầu-cuối (End-to-End Walkthrough)

1. Dựng dự án và biên dịch mã nguồn (Scaffolding & Compiler Pipeline)

  1. Khởi tạo: Chạy lệnh npx create-next-app saas --ts --eslint --tailwind --no-src-dir --no-app. Node.js nạp khung dự án Next.js cấu hình Pages Router, TypeScript, Tailwind CSS.
  2. Biên dịch: Khi deploy lên Vercel:
    • Mã Next.js frontend (TypeScript .ts, .tsx) đi qua trình biên dịch (SWC/Babel) để thực hiện quá trình Transpiling thành vanilla JavaScript. Đồng thời, quá trình Bundling thu gọn tài nguyên vào các file chunk tĩnh để lưu trữ tại các Edge Network (CDN).
    • Thư mục /api được Vercel tự động map độc lập, sử dụng gói @vercel/python cài đặt các dependencies từ requirements.txt để tạo thành serverless Lambda functions.

2. Thứ tự gọi hàm và truyền nhận dữ liệu cấp hàm

  1. Khởi tạo Client: Người dùng truy cập trang chủ /, Next.js load component Home() tại pages/index.tsx. State idea được khởi tạo bằng chuỗi mặc định '...loading'.
  2. Bắt sự kiện kết nối: Sau khi DOM được dựng xong, hook useEffect() tự động kích hoạt một lần duy nhất tại trình duyệt. Nó tạo kết nối EventSource: const evt = new EventSource('/api'); Trình duyệt gửi HTTP GET request với header Accept: text/event-stream tới server.
  3. Xử lý tại Backend (FastAPI):
    • Vercel Router điều hướng yêu cầu tới serverless function của api/index.py. FastAPI tiếp nhận request và gọi hàm xử lý def idea().
    • Hàm idea() khởi tạo client = OpenAI(), tự động đọc biến môi trường OPENAI_API_KEY.
    • Backend gọi OpenAI API: client.chat.completions.create(model="gpt-5-nano", messages=prompt, stream=True). Tham số stream=True chỉ định OpenAI trả dữ liệu dạng luồng liên tục.
    • FastAPI chạy hàm sinh generator event_stream(): vòng lặp for chunk in stream duyệt qua từng mảnh dữ liệu trả về từ OpenAI. Mảnh văn bản được lấy qua chunk.choices[0].delta.content.
    • Do giao thức Server-Sent Events (SSE) sử dụng dòng trống \n\n để phân tách các message, nếu câu trả lời của AI chứa ký tự xuống dòng \n, nó sẽ làm hỏng kết nối SSE của client. Để giải quyết, backend thực hiện tách dòng bằng text.split("\n") và trả về theo chuẩn: yield f"data: {line}\n", cuối gói gửi thêm yield "\n".
    • Hàm idea() trả về một StreamingResponse(event_stream(), media_type="text/event-stream") giúp giữ cổng kết nối HTTP tiếp tục mở.
  4. Render ở Frontend:
    • Tại trình duyệt, mỗi khi nhận được luồng dữ liệu mới từ backend, sự kiện evt.onmessage kích hoạt.
    • Chương trình tích lũy dữ liệu thô vào biến đệm: buffer += e.data và cập nhật state: setIdea(buffer).
    • State idea thay đổi khiến React tính toán lại Virtual DOM và cập nhật trực tiếp lên DOM thật. Thư viện ReactMarkdown phân tích văn bản thô hiển thị thành cấu trúc HTML hoàn chỉnh mà không cần tải lại toàn bộ trang web.
    • Khi luồng kết thúc hoặc gặp lỗi, sự kiện evt.onerror kích hoạt và gọi evt.close() giải phóng socket kết nối.
Mục 6

Techniques - Kỹ thuật sử dụng

Separation of Concerns (SoC)

Mục đích: Phân tách hoàn toàn phần hiển thị giao diện (Next.js) và phần logic xử lý nặng/bảo mật (FastAPI).

Khi nào dùng: Thiết kế kiến trúc cho tất cả các ứng dụng AI SaaS chuyên nghiệp.

Đánh đổi: Tăng độ phức tạp của hệ thống do phải xây dựng và đồng bộ hóa contract dữ liệu qua tầng API trung gian.

Component-based Architecture

Mục đích: Xây dựng giao diện bằng cách chia nhỏ thành các thành phần độc lập (React Components), tự quản lý State và tái sử dụng linh hoạt.

Khi nào dùng: Thiết kế UI/UX trên React/Next.js.

Đánh đổi: Tránh lạm dụng truyền dữ liệu lòng vòng (prop drilling), cần cấu hình cấu trúc thư mục rõ ràng từ đầu.

CLI-driven Scaffolding

Mục đích: Tự động dựng khung cấu trúc dự án Next.js Pages Router qua dòng lệnh với các flag tối ưu (như --no-app).

Khi nào dùng: Khi bắt đầu khởi tạo dự án web mới.

Đánh đổi: Cần nắm vững ý nghĩa các tham số cấu hình để tránh việc tạo dư thừa file hoặc sai router mong muốn.

SSE Streaming

Mục đích: Truyền dữ liệu dạng luồng thời gian thực một chiều từ server về trình duyệt qua giao thức HTTP chuẩn.

Khi nào dùng: Stream kết quả sinh chữ từ mô hình ngôn ngữ lớn (LLM) để nâng cao trải nghiệm người dùng, giảm cảm giác chờ đợi.

Đánh đổi: Kết nối một chiều, chỉ truyền từ server về client. Cần xử lý ngắt dòng để tránh lỗi định dạng của SSE.

1. api/index.py (FastAPI SSE Streaming Backend)

Khởi tạo FastAPI, kết nối OpenAI và sử dụng StreamingResponse kết hợp generator để đẩy kết quả theo thời gian thực.

Đường dẫn: G:/AIProduction_t6_2026/production/week1/day2.md 
from fastapi import FastAPI  # type: ignore
from fastapi.responses import StreamingResponse  # type: ignore # Nhập lớp phản hồi dạng luồng
from openai import OpenAI  # type: ignore

app = FastAPI()

@app.get("/api")
def idea():
    client = OpenAI() # Khởi tạo client OpenAI (tự động đọc API key)
    prompt = [{"role": "user", "content": "Reply with a new business idea for AI Agents, formatted with headings, sub-headings and bullet points"}]
    
    # Gửi yêu cầu sinh text với cờ stream=True để nhận kết quả dạng luồng
    stream = client.chat.completions.create(model="gpt-5-nano", messages=prompt, stream=True)

    # Định nghĩa generator function để xử lý bất tuần tự
    def event_stream():
        for chunk in stream:
            text = chunk.choices[0].delta.content # Trích xuất phần chữ mới sinh ra ở chunk hiện tại
            if text:
                # SSE quy định ngắt bằng \n\n để hết gói tin. Để tránh lỗi khi text chứa \n, ta tách text
                lines = text.split("\n")
                for line in lines:
                    yield f"data: {line}\n" # Gửi dòng text theo chuẩn SSE
                yield "\n" # Gửi dòng trống để báo hiệu hoàn thành message hiện tại

    # Trả về phản hồi dạng luồng với kiểu dữ liệu chuẩn text/event-stream
    return StreamingResponse(event_stream(), media_type="text/event-stream")

2. pages/index.tsx (React Frontend với SSE & Markdown)

Khởi tạo kết nối EventSource ở trình duyệt, tích lũy dữ liệu vào state và render Markdown động.

Đường dẫn: G:/AIProduction_t6_2026/production/week1/day2.md 
"use client" // Đánh dấu đây là Client Component chạy trên trình duyệt

import { useEffect, useState } from 'react';
import ReactMarkdown from 'react-markdown'; // Thư viện parse markdown thành HTML
import remarkGfm from 'remark-gfm'; // Hỗ trợ định dạng bảng, link trong markdown
import remarkBreaks from 'remark-breaks'; // Hỗ trợ ngắt dòng tự nhiên của văn bản

export default function Home() {
    const [idea, setIdea] = useState('…loading'); // State lưu giữ kết quả hiển thị

    useEffect(() => {
        // Khởi tạo đối tượng lắng nghe Server-Sent Events (SSE) từ API backend
        const evt = new EventSource('/api');
        let buffer = '';

        // Lắng nghe dữ liệu đẩy về từ máy chủ
        evt.onmessage = (e) => {
            buffer += e.data; // Cộng dồn chuỗi văn bản nhận được
            setIdea(buffer);  // Cập nhật trạng thái để React kích hoạt re-render
        };

        // Xử lý khi xảy ra lỗi (ví dụ ngắt luồng hoặc mất kết nối)
        evt.onerror = () => {
            console.error('SSE error, closing');
            evt.close(); // Đóng kết nối để giải phóng socket trên browser
        };

        // Hàm cleanup dọn dẹp kết nối khi component bị unmounted
        return () => { evt.close(); };
    }, []);

    return (
        <main className="min-h-screen bg-gradient-to-br from-blue-50 to-indigo-100 dark:from-gray-900 dark:to-gray-800">
            <div className="container mx-auto px-4 py-12">
                <header className="text-center mb-12">
                    <h1 className="text-5xl font-bold bg-gradient-to-r from-blue-600 to-indigo-600 bg-clip-text text-transparent mb-4">
                        Business Idea Generator
                    </h1>
                    <p className="text-gray-600 dark:text-gray-400 text-lg">
                        AI-powered innovation at your fingertips
                    </p>
                </header>

                <div className="max-w-3xl mx-auto">
                    <div className="bg-white dark:bg-gray-800 rounded-2xl shadow-xl p-8 backdrop-blur-lg bg-opacity-95">
                        {idea === '…loading' ? (
                            <div className="flex items-center justify-center py-12">
                                <div className="animate-pulse text-gray-400">
                                    Generating your business idea...
                                </div>
                            </div>
                        ) : (
                            <div className="markdown-content text-gray-700 dark:text-gray-300">
                                <ReactMarkdown remarkPlugins={[remarkGfm, remarkBreaks]}>
                                    {idea}
                                </ReactMarkdown>
                            </div>
                        )}
                    </div>
                </div>
            </div>
        </main>
    );
}

Mục 9

Pitfalls - Bẫy và Lỗi thường gặp

Các trường hợp dễ thất bại (Failure Modes)

  • Lộ OpenAI API Key: Thực hiện gọi trực tiếp API OpenAI từ mã nguồn React frontend. Người dùng có thể F12 và lấy trộm key dễ dàng trong tab Network.
  • Mất giao diện Tailwind khi khôi phục Markdown: Ghi đè toàn bộ nội dung file styles/globals.css bằng các lớp base mới, làm mất các dòng import lõi của Tailwind ở đầu file.
  • Lãng phí cost API khi đóng tab: Client ngắt kết nối đột ngột giữa chừng nhưng server FastAPI vẫn tiếp tục duy trì vòng lặp gọi OpenAI API vô ích.

Cách phòng ngừa và sửa chữa

  • Tách Backend xử lý key: Luôn gọi API LLM thông qua server backend. Server backend giữ API key trong biến môi trường bảo mật trên cloud.
  • Dán code ở cuối file CSS: Chỉ dán thêm (append) đoạn mã CSS khôi phục Markdown vào phía cuối tệp globals.css, giữ nguyên các dòng import Tailwind mặc định.
  • Đóng stream trên server khi ngắt kết nối: Lồng logic generator trong khối try...finally để khi kết nối TCP đứt (Client closed), khối finally lập tức chạy và gọi stream.close() để ngắt luồng gọi OpenAI.
Mục 10

Flashcards & Quiz - Ôn tập tương tác

Q: Tại sao phải sử dụng từ khóa `"use client"` ở đầu tệp index.tsx trong Next.js Pages Router của buổi học này?

A: Để đảm bảo component này chạy hoàn toàn ở phía client (trình duyệt), cho phép sử dụng các React hooks (useState, useEffect) và thực hiện gọi API trực tiếp từ trình duyệt đến backend Python FastAPI độc lập mà không cần Next.js làm server trung gian.

Q: Sự khác biệt lớn nhất về mặt tính chất dữ liệu giữa Props và State trong React là gì?

A: Props là dữ liệu cấu hình truyền từ cha xuống con và là read-only (chỉ đọc) đối với component nhận. State là dữ liệu nội bộ tự quản lý bên trong component, có tính chất động (mutable) và re-render giao diện khi thay đổi.

Trắc nghiệm nhanh: SSE vs WebSockets

Đối với tác vụ truyền phát dữ liệu text thời gian thực từ mô hình ngôn ngữ lớn (LLM streaming), tại sao Server-Sent Events (SSE) lại được ưa chuộng hơn WebSockets?

Mục 11

Quick Memory - Ôn nhanh cuối bài

Những điểm bắt buộc phải thuộc

  1. Vercel tự động nhận diện và build thư mục /api chứa index.py thành serverless FastAPI function mà không cần file cấu hình `vercel.json`.
  2. Lệnh khởi tạo Next.js Pages Router: npx create-next-app saas --ts --eslint --tailwind --no-src-dir --no-app.
  3. Hàm generator Python sử dụng từ khóa yield thay vì return để trả về dữ liệu streaming từng phần.
  4. Media type bắt buộc khai báo cho StreamingResponse SSE trong FastAPI là text/event-stream.
