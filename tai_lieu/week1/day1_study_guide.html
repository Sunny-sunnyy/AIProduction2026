Mục 4

Core Theory - Lý thuyết cốt lõi

Docx LLM Slide

Khái niệm cơ bản

  • App server - máy chủ ứng dụng: Phần mềm chịu trách nhiệm thực thi mã logic (FastAPI) để xử lý các request (yêu cầu) của người dùng.
  • Serverless deployment - triển khai không máy chủ: Mô hình đám mây trong đó lập trình viên không cần quản lý hạ tầng vật lý, tài nguyên được tự động mở rộng và tự động co về 0 khi không có request.
  • Routing - định tuyến: Bản đồ hóa đường dẫn URL của HTTP request tới các hàm tương ứng trong mã nguồn.

Bối cảnh DevOps & Khóa học

  • Platform engineering - kỹ thuật nền tảng: Xây dựng hạ tầng tự phục vụ nhằm giảm tải việc cấu hình hệ thống cho nhà phát triển.
  • T-shape model - mô hình chữ T: Kiến thức rộng trên nhiều Cloud (Azure, GCP, Docker) nhưng đi sâu đặc biệt vào Amazon AWS.
  • Pay-as-you-go - Trả tiền theo thực tế sử dụng: Mô hình tính phí dịch vụ cloud linh hoạt dựa trên tài nguyên tiêu thụ thực tế.
Giải thích sâu về Serverless đằng sau Vercel

Vercel sử dụng hạ tầng AWS Lambda làm lõi vận hành cho các Vercel Functions. Khi lập trình viên cấu hình "use": "@vercel/python" trong file vercel.json, hệ thống tự động đóng gói mã Python của dự án vào một máy ảo cô lập (sandbox container) sử dụng hệ điều hành Amazon Linux. Mỗi khi nhận HTTP Request, container được kích hoạt để thực thi hàm FastAPI. Nếu không có lưu lượng truy cập trong thời gian nhất định, container sẽ bị giải phóng về 0 để tránh tiêu tốn cost (chi phí) chạy nhàn rỗi.

Quy trình hoạt động chi tiết đầu-cuối (End-to-End Walkthrough)

Giai đoạn 1: Triển khai và chạy ứng dụng cơ bản tĩnh

  1. Khởi chạy Local: Khi chạy local bằng lệnh uvicorn instant:app --reload, ASGI server Uvicorn sẽ quét file instant.py, tìm đối tượng app (được khởi tạo bằng FastAPI()) và lắng nghe HTTP requests tại cổng mặc định 8000. Khi client truy cập http://127.0.0.1:8000/, Uvicorn đóng gói request theo chuẩn ASGI và gọi hàm def instant() được ánh xạ bằng decorator @app.get("/"). Hàm xử lý trả về chuỗi văn bản tĩnh.
  2. Cấu hình Vercel Build: Khi chạy lệnh vercel ., Vercel CLI tải toàn bộ thư mục lên máy chủ. Edge Router của Vercel đọc tệp tin vercel.json.
    • Tầng Builder: Node.js runtime của Vercel sử dụng gói @vercel/python để phân tích instant.py và đóng gói các thư viện phụ thuộc trong requirements.txt thành một AWS Lambda package.
    • Tầng Định tuyến: Regex /(.*) cấu hình định tuyến chuyển toàn bộ request đến tệp tin instant.py.

Giai đoạn 2: Nâng cấp ứng dụng tích hợp OpenAI API

  1. Nạp biến môi trường: Lệnh vercel env add OPENAI_API_KEY cho phép lưu trữ khóa bí mật an toàn trên Vercel Cloud. Khi container serverless khởi động, Vercel tự động inject (tiêm) biến này vào môi trường OS của Lambda.
  2. Thực thi code cấp hàm:
    • Khi HTTP request đi qua lớp Edge Network (và đã tắt Deployment Protection), nó kích hoạt serverless container.
    • Hàm instant() được kích hoạt. Lệnh client = OpenAI() khởi tạo kết nối. Thư viện openai tự động đọc biến môi trường OPENAI_API_KEY từ hệ thống để xác thực.
    • Chương trình gửi request POST dạng JSON tới OpenAI endpoint API bằng câu lệnh client.chat.completions.create(model="gpt-5-nano", messages=messages).
    • Sau khi nhận phản hồi, hàm trích xuất nội dung thông qua response.choices[0].message.content.
    • Hàm thực hiện hàm chuỗi .replace("\n", "<br/>") để định dạng hiển thị xuống dòng trên web, bọc kết quả vào khung HTML và trả về dưới dạng HTMLResponse.

2. instant.py (Phiên bản tích hợp OpenAI API)

Gọi OpenAI API để sinh lời chào mừng ngẫu nhiên và phản hồi dưới dạng HTML động.

from fastapi import FastAPI
from fastapi.responses import HTMLResponse # Nhập HTMLResponse để trả về định dạng HTML thay vì JSON mặc định
from openai import OpenAI # Nhập thư viện client chính thức của OpenAI

app = FastAPI()

@app.get("/", response_class=HTMLResponse) # Chỉ định kiểu phản hồi trả về là HTMLResponse
def instant():
    client = OpenAI() # Khởi tạo client OpenAI. Nó sẽ tự động đọc biến môi trường OPENAI_API_KEY từ hệ điều hành
    message = """
You are on a website that has just been deployed to production for the first time!
Please reply with an enthusiastic announcement to welcome visitors to the site, explaining that it is live on production for the first time!
"""
    messages = [{"role": "user", "content": message}] # Thiết lập bối cảnh hội thoại đầu vào cho mô hình AI
    
    # Gửi request tạo phản hồi chat completion sử dụng model gpt-5-nano (tên model mô phỏng từ bài học)
    response = client.chat.completions.create(model="gpt-5-nano", messages=messages)
    
    # Nhận câu trả lời từ AI và thay thế các ký tự xuống dòng bằng thẻ xuống hàng 
 của HTML
    reply = response.choices[0].message.content.replace("\n", "
")
    
    # Tạo cấu trúc tài liệu HTML cơ bản chứa nội dung chào mừng từ AI
    html = f"Live in an Instant!
{reply}
"
    return html

Mục 9

Pitfalls - Bẫy và Lỗi thường gặp

Các trường hợp dễ thất bại (Failure Modes)

  • Lỗi vercel: command not found: Lệnh cài đặt Vercel CLI toàn cục qua npm thành công nhưng gõ lệnh terminal không nhận diện.
  • Bất tương thích kiến trúc hệ thống (System architecture mismatch): Triển khai các thư viện Python biên dịch bằng nhân C/C++ (NumPy, Pydantic bản mới) gặp lỗi ImportError bí ẩn khi chạy trên AWS Lambda.
  • Lạm dụng sửa lỗi bề mặt của LLM (Band-aid fixes): Copy-paste toàn bộ log lỗi dán vào LLM và đổi code ứng dụng liên tục trong khi lỗi thực chất nằm ở cấu hình môi trường cloud.

Cách phòng ngừa và sửa chữa

  • Khởi động lại Terminal: Hãy tắt terminal hiện tại trong Cursor và mở một cái mới để hệ điều hành nạp lại biến môi trường PATH chứa thư mục npm toàn cục.
  • Build qua Docker: Sử dụng môi trường ảo hóa Docker chạy hệ điều hành Linux (tương đồng với Vercel/Lambda) để thực hiện build dependencies và đóng gói trước khi tải lên Cloud.
  • Phân tích nguyên nhân gốc rễ (Root Cause Analysis): Luôn đặt câu hỏi ngược lại cho LLM: "Giải pháp sửa lỗi này có ảnh hưởng tới môi trường cloud không? Nguyên nhân gốc rễ gây ra lỗi này là gì?"
Mục 10

Flashcards & Quiz - Ôn tập tương tác

Q: Tại sao phải vô hiệu hóa Deployment Protection trên Dashboard của Vercel?

A: Vercel mặc định bật tính năng này để bảo vệ các phiên bản preview (thử nghiệm), yêu cầu người dùng phải đăng nhập tài khoản Vercel team mới xem được. Việc tắt nó đi giúp ứng dụng SaaS của bạn có thể truy cập công khai bởi bất kỳ ai trên internet.

Q: Lệnh để cài đặt Vercel CLI toàn cục trên máy tính là gì?

A: Chạy câu lệnh npm install -g vercel trong terminal. Yêu cầu máy tính đã cài đặt sẵn môi trường chạy Node.js trước đó.

Mục 11

Quick Memory - Ôn nhanh cuối bài

Những điểm bắt buộc phải thuộc

  1. FastAPI chạy ở môi trường phát triển local luôn cần một ASGI server như Uvicorn để thực thi.
  2. Vercel tự quản lý runtime engine trên môi trường đám mây bằng serverless functions, không cần Uvicorn tại production.
  3. Thư viện openai tự động xác thực bằng biến môi trường có tên chính xác là OPENAI_API_KEY.
  4. Lệnh vercel --prod dùng để đẩy phiên bản code hiện tại lên môi trường sản xuất (production) chính thức, thay thế bản build preview trước đó.
