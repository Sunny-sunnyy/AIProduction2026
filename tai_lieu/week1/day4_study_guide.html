Core Theory - Lý thuyết cốt lõi

Slide Transcript

Key Concepts - Khái niệm chính

  • Pydantic BaseModel - mô hình cơ sở Pydantic: Một class của thư viện Pydantic trong Python, đóng vai trò định hình lược đồ (schema) dữ liệu và tự động kiểm chứng kiểu (type checking) của request body.
  • Structured Prompt - gợi ý có cấu trúc: Kỹ thuật phân tách rõ nhiệm vụ hệ thống (System Prompt) và dữ liệu lâm sàng cụ thể (User Prompt) để điều khiển hành vi của LLM, giúp định dạng đầu ra của AI đồng bộ và ổn định.
  • Electronic Health Record (EHR) - hồ sơ bệnh án điện tử: Hệ thống phần mềm y khoa lưu trữ lịch sử sức khỏe bệnh nhân dạng số hóa. Đây là đích đến cuối cùng của các tài liệu lâm sàng do bác sĩ tạo ra.
  • Platform as a Service (PaaS) - nền tảng như một dịch vụ: Mô hình cloud cung cấp sẵn runtime và cơ chế build tự động (ví dụ: Vercel). Giúp lập trình viên chỉ cần đẩy code mà không cần cấu hình máy chủ.
  • Infrastructure as a Service (IaaS) - hạ tầng như một dịch vụ: Mô hình cloud cung cấp phần cứng thô (ví dụ: AWS). Đòi hỏi tự cài đặt Docker containerization, quản lý phân quyền IAM và hạ tầng mạng tự động co giãn.

PaaS vs IaaS Checkpoint

Bài giảng số 22 chỉ ra một triết lý thiết kế SaaS AI quan trọng: Bản thân việc gọi API LLM chỉ là một tính năng phụ (side note). Phần lớn công sức nằm ở kỹ nghệ hạ tầng (Platform Engineering) và DevOps: làm thế nào để đóng gói Docker, phân quyền cổng mạng bảo mật và co giãn tự động trên AWS.

Chuyển từ Vercel (PaaS) sang AWS (IaaS) ở Day 5 mang lại cho ứng dụng SaaS khả năng tùy biến vô hạn, bảo mật cấp doanh nghiệp và kiểm soát chi phí tối ưu khi phục vụ hàng triệu bác sĩ khám bệnh hàng ngày.

Deep Explanation - Tại sao Pydantic Validation lại là "Tường lửa" của API?

Trong lập trình API truyền thống bằng Python, việc tiếp nhận dữ liệu JSON yêu cầu lập trình viên phải viết rất nhiều dòng code thủ công để kiểm tra khóa tồn tại: if 'patient_name' not in data: raise ValueError và ép kiểu dữ liệu. Điều này dẫn đến code rườm rà, dễ bỏ sót lỗi và khó bảo trì.

FastAPI giải quyết triệt để vấn đề này bằng cách tích hợp sâu thư viện Pydantic. Khi bạn khai báo một class kế thừa từ BaseModel:

  • FastAPI sẽ tự động phân tích cú pháp (parse) JSON payload gửi lên trong request body.
  • Kiểm tra xem dữ liệu có thiếu các trường bắt buộc hay sai kiểu dữ liệu (như gửi số thay vì chuỗi) không.
  • Nếu phát hiện sai lệch, FastAPI lập tức chặn đứng request ở ngay "cửa ngõ" API và trả về mã lỗi HTTP 422 Unprocessable Entity kèm thông tin chi tiết lỗi, bảo vệ mã nguồn nghiệp vụ y khoa phía sau không bị crash do thiếu tham số.

Workflow / Pipeline - Quy trình hoạt động đầu-cuối

Code Slide

High-level Flow - Luồng tổng quát

  1. Nhập liệu: Bác sĩ nhập tên bệnh nhân, chọn ngày khám quaDatepicker, và nhập ghi chú khám lâm sàng thô.
  2. Đóng gói & Gửi POST: Frontend chuyển dữ liệu thành đối tượng JSON và gọi phương thức POST qua fetchEventSource('/api') đính kèm token JWT.
  3. Validate Backend: FastAPI nhận payload, gán vào model Pydantic Visit để validate kiểu dữ liệu.
  4. Tạo prompt cấu trúc: Backend ghép dữ liệu lâm sàng của Visit vào template prompt chia rõ vai trò System/User.
  5. Stream OpenAI: Gọi API OpenAI với stream=True và dùng hàm event_stream() để yield dữ liệu SSE trả về.

Data & Control Flow - Luồng dữ liệu / điều khiển

Mô hình y khoa này chuyển từ kết nối GET sang POST để có thể truyền tải các khối văn bản ghi chú lâm sàng cực lớn của bác sĩ một cách an toàn và không bị giới hạn độ dài ký tự của URL. Đầu ra của OpenAI được cấu trúc hóa định hướng chặt chẽ bằng System Prompt để luôn trả về 3 phần phân định rõ ràng bằng thẻ Markdown.

Section 6

Techniques - Kỹ thuật sử dụng

1. Pydantic Data Validation

Purpose - Mục đích: Tự động xác thực và ép kiểu dữ liệu JSON payload gửi từ Next.js lên backend FastAPI, đảm bảo code nghiệp vụ không bị crash vì thiếu tham số.

When to use - Dùng khi nào: Dùng cho tất cả các API nhận dữ liệu đầu vào phức tạp từ biểu mẫu form.

Trade-off - Đánh đổi: Phải viết thêm class Pydantic định nghĩa cấu trúc dữ liệu tĩnh rõ ràng, làm giảm tính tùy biến dữ liệu động linh hoạt.

2. Structured Prompts

Purpose - Mục đích: Hướng LLM phân chia đầu ra y khoa thành chính xác 3 phần (Tóm tắt bệnh án, Hành động của bác sĩ, Email bệnh nhân) để nâng cao trải nghiệm ứng dụng.

When to use - Dùng khi nào: Khi đầu ra của mô hình AI cần được định dạng chính xác để hiển thị hoặc xử lý tiếp theo.

Trade-off - Đánh đổi: Phụ thuộc vào chất lượng hiểu chỉ thị của mô hình LLM. Với các mô hình nhỏ, có thể đôi khi AI sinh sai số lượng đề mục.

3. local validation before deploy

Purpose - Mục đích: Rà soát và cập nhật đầy đủ các file cấu hình thư viện y khoa (như file requirements.txt chứa pydantic) trước khi chạy lệnh deploy của Vercel.

When to use - Dùng khi nào: Thực hiện trước mỗi lần cập nhật sản phẩm lên môi trường live production.

Trade-off - Đánh đổi: Tốn thêm thời gian kiểm tra thủ công môi trường ảo (virtualenv) và file cấu hình trước khi deploy.

4. Demo Sandbox Labeling

Purpose - Mục đích: Hiển thị rõ ràng dòng chữ "For demonstration purposes only" trên giao diện để hạn chế tối đa các rủi ro pháp lý y tế ngoài ý muốn.

When to use - Dùng khi nào: Triển khai các ứng dụng y khoa, tài chính ở giai đoạn demo công cộng.

Trade-off - Đánh đổi: Làm giảm bớt độ "thực tế" của sản phẩm trong mắt khách hàng trải nghiệm thử.

File: api/index.py

Backend FastAPI POST endpoint nhận và validate dữ liệu y tế qua Pydantic, sử dụng prompt có cấu trúc để sinh bệnh án.

FastAPI Backend Pydantic Model Structured Prompts 
import os
from fastapi import FastAPI, Depends  # type: ignore
from fastapi.responses import StreamingResponse  # type: ignore
from pydantic import BaseModel  # type: ignore
from fastapi_clerk_auth import ClerkConfig, ClerkHTTPBearer, HTTPAuthorizationCredentials  # type: ignore
from openai import OpenAI  # type: ignore

app = FastAPI()
clerk_config = ClerkConfig(jwks_url=os.getenv("CLERK_JWKS_URL"))
clerk_guard = ClerkHTTPBearer(clerk_config)

# Bước 1: Khai báo cấu trúc dữ liệu y khoa bằng Pydantic model
class Visit(BaseModel):
    patient_name: str
    date_of_visit: str
    notes: str

# Bước 2: Dựng system prompt cố định cấu trúc đầu ra gồm 3 phần
system_prompt = """
You are provided with notes written by a doctor from a patient's visit.
Your job is to summarize the visit for the doctor and provide an email.
Reply with exactly three sections with the headings:
### Summary of visit for the doctor's records
### Next steps for the doctor
### Draft of email to patient in patient-friendly language
"""

# Bước 3: Định dạng user prompt động từ đối tượng Visit
def user_prompt_for(visit: Visit) -> str:
    return f"""Create the summary, next steps and draft email for:
Patient Name: {visit.patient_name}
Date of Visit: {visit.date_of_visit}
Notes:
{visit.notes}"""

@app.post("/api") # Đổi sang POST để nhận JSON payload
def consultation_summary(
    visit: Visit, # FastAPI tự động ép kiểu và validate JSON nhận được thành đối tượng Visit
    creds: HTTPAuthorizationCredentials = Depends(clerk_guard),
):
    user_id = creds.decoded["sub"]  # Lấy ID của bác sĩ đăng nhập phục vụ audit log
    client = OpenAI()

    user_prompt = user_prompt_for(visit)

    prompt = [
        {"role": "system", "content": system_prompt},
        {"role": "user", "content": user_prompt},
    ]

    stream = client.chat.completions.create(
        model="gpt-5-nano",
        messages=prompt,
        stream=True,
    )

    # Đóng gói luồng SSE gửi về client Next.js
    def event_stream():
        for chunk in stream:
            text = chunk.choices[0].delta.content
            if text:
                lines = text.split("\n")
                for line in lines[:-1]:
                    yield f"data: {line}\n\n"
                    yield "data:  \n"
                yield f"data: {lines[-1]}\n\n"

    return StreamingResponse(event_stream(), media_type="text/event-stream")

import { useState, FormEvent } from 'react';
import { useAuth } from '@clerk/nextjs';
import DatePicker from 'react-datepicker';

import remarkBreaks from 'remark-breaks';
import { fetchEventSource } from '@microsoft/fetch-event-source';

function ConsultationForm() {
    const { getToken } = useAuth();

    // Khởi tạo trạng thái form nhập liệu y khoa
    const [patientName, setPatientName] = useState('');
    const [visitDate, setVisitDate] = useState<Date | null>(new Date());
    const [notes, setNotes] = useState('');

    const [output, setOutput] = useState('');
    const [loading, setLoading] = useState(false);

    async function handleSubmit(e: FormEvent) {
        e.preventDefault(); // Chặn hành động reload trang của form HTML mặc định
        setOutput('');
        setLoading(true);

        const jwt = await getToken();
        if (!jwt) {
            setOutput('Authentication required');
            setLoading(false);
            return;
        }

        const controller = new AbortController();
        let buffer = '';

        // Thực hiện cuộc gọi API POST truyền dữ liệu JSON
        await fetchEventSource('/api', {
            signal: controller.signal,
            method: 'POST', // Ép phương thức gửi thành POST
            headers: {
                'Content-Type': 'application/json',
                Authorization: `Bearer ${jwt}`,
            },
            body: JSON.stringify({
                patient_name: patientName,
                // Định dạng ngày khám thành chuỗi yyyy-mm-dd
                date_of_visit: visitDate?.toISOString().slice(0, 10),
                notes,
            }),
            onmessage(ev) {
                buffer += ev.data;
                setOutput(buffer); // Nối chuỗi cập nhật Markdown liên tục
            },
            onclose() { 
                setLoading(false); 
            },
            onerror(err) {
                console.error('SSE error:', err);
                controller.abort();
                setLoading(false);
            },
        });
    }

    return (
        <div className="container mx-auto px-4 py-12 max-w-3xl">
            <h1 className="text-4xl font-bold text-gray-900 dark:text-gray-100 mb-8">
                Consultation Notes
            </h1>

            <form onSubmit={handleSubmit} className="space-y-6 bg-white dark:bg-gray-800 rounded-xl shadow-lg p-8">
                {/* Trường nhập tên bệnh nhân */}
                <div className="space-y-2">
                    <label htmlFor="patient" className="block text-sm font-medium text-gray-700 dark:text-gray-300">
                        Patient Name
                    </label>
                    <input
                        id="patient"
                        type="text"
                        required
                        value={patientName}
                        onChange={(e) => setPatientName(e.target.value)}
                        className="w-full px-4 py-2 border border-gray-300 dark:border-gray-600 rounded-lg dark:bg-gray-700 dark:text-white"
                        placeholder="Enter patient's full name"
                    />
                </div>

                {/* Linh kiện chọn ngày khám */}
                <div className="space-y-2">
                    <label htmlFor="date" className="block text-sm font-medium text-gray-700 dark:text-gray-300">
                        Date of Visit
                    </label>
                    <DatePicker
                        id="date"
                        selected={visitDate}
                        onChange={(d: Date | null) => setVisitDate(d)}
                        dateFormat="yyyy-MM-dd"
                        placeholderText="Select date"
                        required
                        className="w-full px-4 py-2 border border-gray-300 dark:border-gray-600 rounded-lg dark:bg-gray-700 dark:text-white"
                    />
                </div>

                {/* Trường ghi chú khám lâm sàng thô */}
                <div className="space-y-2">
                    <label htmlFor="notes" className="block text-sm font-medium text-gray-700 dark:text-gray-300">
                        Consultation Notes
                    </label>
                    <textarea
                        id="notes"
                        required
                        rows={8}
                        value={notes}
                        onChange={(e) => setNotes(e.target.value)}
                        className="w-full px-4 py-2 border border-gray-300 dark:border-gray-600 rounded-lg dark:bg-gray-700 dark:text-white"
                        placeholder="Enter detailed consultation notes..."
                    />
                </div>

                <button 
                    type="submit" 
                    disabled={loading}
                    className="w-full bg-blue-600 hover:bg-blue-700 disabled:bg-blue-400 text-white font-semibold py-3 px-6 rounded-lg transition-colors"
                >
                    {loading ? 'Generating Summary...' : 'Generate Summary'}
                </button>
            </form>

            {/* Khu vực hiển thị kết quả bệnh án AI */}
            {output && (
                <section className="mt-8 bg-gray-50 dark:bg-gray-800 rounded-xl shadow-lg p-8">
                    <div className="markdown-content prose prose-blue dark:prose-invert max-w-none">
                        <ReactMarkdown remarkPlugins={[remarkGfm, remarkBreaks]}>
                            {output}
                        </ReactMarkdown>
                    </div>
                </section>
            )}
        </div>
    );
}

Options / Trade-offs - Bản đồ lựa chọn

Lựa chọn kỹ thuật Phù hợp nhất khi Đánh đổi (Trade-off) Rủi ro chính (Risk)
Kết quả dạng Markdown thô (chọn trong bài học) Giai đoạn thử nghiệm MVP để kiểm chứng nhanh độ hữu ích của bệnh án AI. Khó bóc tách tự động các trường dữ liệu để đưa vào database. LLM có thể định dạng sai lệch hoặc thiếu đề mục nếu prompt bị nhiễu.

Triển khai trên Vercel PaaS Ứng dụng MVP, dự án cá nhân, cần triển khai nhanh gọn để giới thiệu sản phẩm. Giới hạn thời gian chạy Serverless Function và chi phí tăng nhanh khi scale. Bị ràng buộc bởi cấu hình mặc định của Vercel (ví dụ: timeout tối đa 60 giây).
Triển khai trên AWS IaaS (Day 5) Sản phẩm SaaS đi vào hoạt động chính thức, bắt đầu có khách hàng trả phí lớn (B2B SaaS). Tốn rất nhiều công sức thiết lập hạ tầng mạng, IAM, Docker container lúc đầu. Chi phí phát sinh lớn nếu cấu hình sai tài nguyên cloud chạy không tải.
Section 9

Pitfalls - Lỗi / bẫy thường gặp

Failure Modes - Các trường hợp lỗi

  • Lỗi HTTP 405 Method Not Allowed: Console báo lỗi đỏ và API ngắt kết nối ngay khi bấm Generate. Lý do: Frontend Next.js gửi request dạng GET trong khi backend FastAPI đã đổi sang decorator POST (hoặc ngược lại).
  • Datepicker bị vỡ giao diện: Giao diện chọn ngày tháng của Datepicker hiển thị dạng text thô kéo dài, đè nát bố cục trang. Lý do: Quên import file CSS giao diện của Datepicker.
  • Lỗi Server Error (HTTP 500) khi live: Ứng dụng chạy tốt ở localhost nhưng khi deploy lên Vercel live thì gọi API lỗi 500. Lý do: Quên khai báo thư viện pydantic trong file requirements.txt.

Fix / Prevention - Cách sửa đổi

  • Khắc phục lỗi 405: Đảm bảo cấu hình method: 'POST' trong fetchEventSource đồng bộ với decorator @app.post("/api") của FastAPI.
  • Khắc phục Datepicker vỡ layout: Thêm dòng import 'react-datepicker/dist/react-datepicker.css'; vào file cấu hình gốc pages/_app.tsx.
  • Khắc phục lỗi 500 Vercel: Luôn rà soát và ghi đầy đủ mọi dependencies mới (fastapi, pydantic, openai) vào file requirements.txt trước khi chạy lệnh vercel --prod.
Section 10

Flashcards & Quiz - Thẻ ôn tập tương tác

Q: Tại sao phải chuyển API từ GET sang POST khi tích hợp ứng dụng y tế?

A: Để chuyển tải payload JSON chứa dữ liệu khám bệnh phức tạp. Giao thức GET chỉ hỗ trợ truyền tham số qua URL query parameter (bị giới hạn độ dài ký tự và không an toàn cho thông tin bệnh nhân). POST hỗ trợ truyền tải nội dung văn bản lớn trong request body một cách bảo mật.

Q: Sự khác biệt cơ bản giữa Vercel (PaaS) và AWS (IaaS) là gì?

A: Vercel (PaaS) tự động hóa hoàn toàn việc build và deploy, lập trình viên không cần biết hệ điều hành phía sau là gì. AWS (IaaS) cung cấp hạ tầng thô, lập trình viên phải tự cấu hình phân quyền mạng (VPC), máy chủ ảo, ECR, và Docker container nhưng đổi lại có sự kiểm soát tuyệt đối về bảo mật và chi phí.
