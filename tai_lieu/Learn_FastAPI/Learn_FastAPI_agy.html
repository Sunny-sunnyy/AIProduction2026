Antigravity FastAPI

1. FastAPI Essentials & Overview - Kiến thức cơ bản & Tổng quan về FastAPI

FastAPI là một web framework - khung công việc web hiện đại, hiệu năng cao để xây dựng các APIs (giao diện lập trình ứng dụng) với Python 3.8+ dựa trên standard Python type hints - gợi ý kiểu tiêu chuẩn của Python. Được tạo bởi Sebastián Ramírez vào năm 2018, nó đã nhanh chóng trở thành một trong các framework (khung công việc) tiêu chuẩn cho phát triển backend (phía máy chủ), đặc biệt trong data science - khoa học dữ liệu, machine learning - học máy và các microservices (dịch vụ siêu nhỏ) có độ đồng thời cao.

Core Features & Advantages - Các tính năng cốt lõi & Ưu điểm

Khi nghiên cứu FastAPI, điều cần thiết là phải hiểu tại sao nó được ưa chuộng hơn các framework (khung công việc) kế thừa như Flask hoặc Django cho việc phát triển API-first development - phát triển ưu tiên API:

  • High Performance - Hiệu năng cao: FastAPI là một trong những Python framework - khung công việc Python nhanh nhất hiện nay, ngang hàng với NodeJS và Go. Tốc độ này đạt được nhờ tận dụng Starlette (cho routing (định tuyến) web và các tính năng) and Uvicorn (một ASGI server - máy chủ ASGI).
  • Asynchronous Support - Hỗ trợ bất đồng bộ: Nó hỗ trợ tự nhiên lập trình bất đồng bộ Python hiện đại (sử dụng async và await), cho phép server (máy chủ) xử lý hàng ngàn request (yêu cầu) đồng thời một cách hiệu quả.
  • Automatic Documentation - Tài liệu tự động: Theo mặc định, FastAPI tự động tạo automatic interactive API documentation - tài liệu API tương tác tự động. Ngay khi cài đặt, bạn có quyền truy cập vào Swagger UI - giao diện kiểm thử tương tác tại /docs and ReDoc (tài liệu cấu trúc sạch sẽ tại /redoc).
  • Type Safety & Validation - An toàn kiểu dữ liệu & Xác thực: Được vận hành bởi Pydantic, FastAPI thực hiện data validation - xác thực dữ liệu đối với request (yêu cầu) đầu vào và thực hiện data serialization - tuần tự hóa dữ liệu đối với response (phản hồi) đầu ra dựa trên standard Python type hints - gợi ý kiểu tiêu chuẩn của Python. Các request (yêu cầu) không hợp lệ sẽ tự động bị từ chối với các phản hồi lỗi rõ ràng và có cấu trúc.
  • Developer Productivity - Năng suất của nhà phát triển: Bằng cách sử dụng type hints - gợi ý kiểu, các trình soạn thảo mã nguồn cung cấp tính năng tự động hoàn thành (autocompletion) và kiểm tra lỗi (linting) cho tất cả các path parameter - tham số đường dẫn, query parameter - tham số truy vấn và request body - thân yêu cầu. Điều này giúp tăng tốc độ phát triển từ 200% đến 300% và giảm tới 40% lỗi do con người tạo ra.

Note (Lưu ý): Thực chất ở bên dưới, FastAPI không phải là một web framework - khung công việc web hoàn toàn mới được viết từ đầu. Nó là một lớp bọc mỏng nhưng mạnh mẽ, kết hợp tốc độ thô và khả năng routing (định tuyến) web của Starlette với các tính năng data validation - xác thực dữ liệu và data serialization - tuần tự hóa dữ liệu mạnh mẽ của Pydantic.

Việc chọn đúng Python framework - khung công việc Python phụ thuộc vào các yêu cầu dự án của bạn. Dưới đây là bảng so sánh cấu trúc chỉ ra những điểm FastAPI vượt trội so với các framework (khung công việc) kế thừa:

Mặc dù công cụ tích hợp sẵn venv (môi trường ảo) của Python hoạt động ổn định, các đội ngũ phát triển hiện đại ưa chuộng uv từ Astral, một trình cài đặt và giải quyết gói siêu nhanh viết bằng Rust hoạt động như một sự thay thế trực tiếp cho pip và virtualenv.

  • environment management - quản lý môi trường: Nó tự động xử lý việc tạo và kích hoạt môi trường ảo.

3. Core Concepts & Fundamentals - Các Khái niệm Cốt lõi & Cơ bản

Để làm chủ FastAPI, bạn phải hiểu các thành phần xây dựng cốt lõi của nó: request routing - định tuyến yêu cầu, parameter validation - xác thực tham số, request-response pipeline - đường ống yêu cầu-phản hồi (hay request/response lifecycle - chu kỳ sống yêu cầu/phản hồi), data schemas - sơ đồ dữ liệu và dependency injection - tiêm phụ thuộc.

Hello World Application - Ứng dụng Hello World

Tạo một API cơ bản rất đơn giản và đòi hỏi rất ít boilerplate (mã nguồn khuôn mẫu). Dưới đây là một ví dụ hoạt động tối giản trong main.py:

from fastapi import FastAPI

# 1. Create a FastAPI instance
app = FastAPI()

# 2. Define a path operation decorator
@app.get("/")
# 3. Define an asynchronous path operation function
async def read_root():
    # 4. Return a dictionary (automatically serialized to JSON)
    return {"message": "Hello World"}

Line-by-Line Breakdown - Phân tích Từng Dòng:

  1. from fastapi import FastAPI: Nhập lớp FastAPI chính quản lý routing (định tuyến), các sự kiện và cấu hình ứng dụng.
  2. app = FastAPI(): Khởi tạo ứng dụng. Đối tượng này điều phối tất cả các request (yêu cầu), tài liệu và dependency (phụ thuộc).
  3. @app.get("/"): Một path operation decorator - bộ trang trí hoạt động đường dẫn. Nó ánh xạ các HTTP GET request (yêu cầu) trên đường dẫn / tới hàm bên dưới.
  4. async def read_root(): Một asynchronous path operation function - hàm hoạt động đường dẫn bất đồng bộ. Sử dụng async def cho phép các hoạt động non-blocking - không chặn. Bạn cũng có thể viết def read_root() cho các endpoint (điểm cuối) đồng bộ; FastAPI xử lý cả hai một cách thông minh.
  5. return {"message": "Hello World"}: Trả về một từ điển Python (Python dict). FastAPI tự động xử lý data serialization - tuần tự hóa dữ liệu của nó sang JSON và đặt tiêu đề Content-Type thành application/json.

Routing: Path vs. Query Parameters - Định tuyến: Tham số đường dẫn so với Tham số truy vấn

FastAPI phân tách các tham số yêu cầu thành hai loại chính dựa trên vị trí xuất hiện của chúng trong URL:

  • Path Parameters - Tham số đường dẫn: Là một phần của chính đường dẫn URL, được sử dụng để xác định các tài nguyên cụ thể. Ví dụ, trong /items/42, số 42 là một path parameter - tham số đường dẫn.
  • Query Parameters - Tham số truy vấn: Các tham số tùy chọn được thêm vào sau dấu ? trong URL, được sử dụng để lọc, sắp xếp hoặc phân trang. Ví dụ, trong /items?limit=10, limit là một query parameter - tham số truy vấn.

Parameter Validation - Xác thực Tham số

Chúng ta có thể áp đặt các ràng buộc lên các tham số này bằng cách sử dụng các đối tượng Path và Query, các đối tượng này cũng sẽ điền mô tả và siêu dữ liệu xác thực vào tài liệu được tạo tự động:

from fastapi import FastAPI, Path, Query
from typing import Optional

app = FastAPI()

@app.get("/items/{item_id}")
async def read_item(
    # Validation: item_id must be an integer greater than 0 (gt=0)
    item_id: int = Path(..., description="The ID of the item to retrieve", gt=0),
    # Validation: q is optional, min_length of 3, max_length of 50
    q: Optional[str] = Query(None, min_length=3, max_length=50, description="Search query string")
):
    return {"item_id": item_id, "query": q}

@app.get("/users")
async def list_users(
    # Validation: limit has default 10, must be between 1 and 100 (ge=1, le=100)
    limit: int = Query(10, ge=1, le=100, description="Limit the number of returned users"),
    active: bool = Query(True, description="Filter users by active status")
):
    return {"limit": limit, "active": active}

Pydantic Schemas - Sơ đồ Pydantic

Pydantic là validation engine - công cụ xác thực của FastAPI. Bằng cách định nghĩa các lớp kế thừa từ BaseModel của Pydantic, bạn khai báo cấu trúc dữ liệu của mình. Điều này cho phép tự động phân tích request body - thân yêu cầu, xác thực và tạo schema (sơ đồ). Trong các ứng dụng production (môi trường vận hành thực tế), việc phân tách các schema (sơ đồ) dựa trên mục đích sử dụng là một mẫu thiết kế tiêu chuẩn:

  • Base Schema - Sơ đồ cơ sở: Chứa các thuộc tính chung được chia sẻ giữa việc tạo mới và truy xuất.
  • Create Schema - Sơ đồ tạo mới: Mở rộng sơ đồ cơ sở với các thuộc tính chỉ ghi (chẳng hạn như mật khẩu văn bản thô).
  • Response Schema - Sơ đồ phản hồi: Lọc bỏ các thuộc tính nhạy cảm (chẳng hạn như mật khẩu hoặc các khóa nội bộ) trước khi trả lại dữ liệu cho client (máy khách).
from pydantic import BaseModel, Field
from typing import Optional

# Base schema containing shared attributes
class UserBase(BaseModel):
    username: str = Field(..., min_length=3, max_length=50)
    email: str # In production, use pydantic.EmailStr (requires email-validator)

# Schema used for user registration
class UserCreate(UserBase):
    password: str = Field(..., min_length=8, description="User's plain password")

# Schema returned to the client (excludes password fields)
class User(UserBase):
    id: int
    is_active: bool

    class Config:
        # In Pydantic v2, from_attributes=True replaces orm_mode=True
        # This enables deserialization from ORM instances (e.g., SQLAlchemy models)
        from_attributes = True

Request Body & Response Model - Thân yêu cầu & Mô hình phản hồi

Khi triển khai các endpoint (điểm cuối) định tuyến, chúng ta khai báo các kiểu request body - thân yêu cầu bằng cách sử dụng các Pydantic schemas - sơ đồ Pydantic của mình, và cấu hình serialization (tuần tự hóa) với tham số response_model bên trong route decorator - bộ trang trí định tuyến:

from fastapi import FastAPI, HTTPException, status

app = FastAPI()

# Dummy database storage
db_users = []

# Declaring response_model filters outgoing data, ensuring no password is leaked
@app.post("/users", response_model=User, status_code=status.HTTP_201_CREATED)
async def create_user(user_in: UserCreate):
    # Validate if email is already taken
    for u in db_users:
        if u["email"] == user_in.email:
            raise HTTPException(status_code=400, detail="Email already registered")
            
    # Save user (normally password is hashed here)
    user_dict = user_in.model_dump() # In Pydantic v2, .model_dump() replaces .dict()
    user_dict["id"] = len(db_users) + 1
    user_dict["is_active"] = True
    
    # Store user database record (which contains password)
    db_users.append(user_dict)
    
    # Returning the dictionary. FastAPI serializes it using the User response schema,
    # which excludes the 'password' field.
    return user_dict

Dependency Injection - Tiêm phụ thuộc (DI)

Dependency injection - tiêm phụ thuộc (DI) là một mẫu thiết kế phần mềm trong đó các đối tượng hoặc hàm nhận các phụ thuộc khác mà chúng yêu cầu. Trong FastAPI, hệ thống DI được tích hợp sẵn và mạnh mẽ, cho phép các endpoint (điểm cuối) khai báo và nhận tài nguyên một cách động bằng cách sử dụng tiện ích Depends().

Key Benefits of Dependency Injection in FastAPI - Các lợi ích chính của Dependency Injection trong FastAPI:

  • Lifecycle Management - Quản lý chu kỳ sống: Lý tưởng để quản lý các tài nguyên như database session - phiên cơ sở dữ liệu, các kết nối mạng hoặc tệp tin. Hệ thống DI tự động xử lý việc khởi tạo và dọn dẹp.
  • Modularity & Reuse - Tính mô-đun & Tái sử dụng: Viết mã một lần (ví dụ: logic xác thực hoặc truy xuất cơ sở dữ liệu) và tái sử dụng nó trên nhiều endpoint (điểm cuối).
  • Seamless Testing - Kiểm thử liền mạch: Dễ dàng hoán đổi các phụ thuộc bằng các triển khai giả lập (mock) trong môi trường kiểm thử (sử dụng app.dependency_overrides).

Dưới đây là một ví dụ về việc tiêm một database session - phiên cơ sở dữ liệu phụ thuộc, cho thấy cách FastAPI xử lý cả việc khởi tạo (yield) và dọn dẹp (đóng phiên) tài nguyên:

from fastapi import FastAPI, Depends
from sqlalchemy.orm import Session
# Dummy database session for representation
# from database import SessionLocal, Item

app = FastAPI()

# 1. Define a dependency function managing database session lifecycle
def get_db():
    db = SessionLocal() # Establish database session
    try:
        yield db       # Inject the session into the endpoint function
    finally:
        db.close()     # Executed AFTER the endpoint returns response

# 2. Inject the dependency into path operation functions
@app.get("/items")
def read_items(db: Session = Depends(get_db)):
    # The session is active and safe to use
    items = db.query(Item).all()
    return items

Roles of Core Modules - Vai trò của các Mô-đun Cốt lõi:

  • core/: Tập trung các cấu hình hệ thống thông qua Pydantic BaseSettings (ví dụ: đọc thông tin xác thực cơ sở dữ liệu và các nguồn gốc CORS từ .env) và chứa các tiện ích trợ giúp JWT.
  • db/: Khởi tạo công cụ SQLAlchemy, cấu hình SessionLocal (nhà máy tạo phiên) và xuất ra dependency (phụ thuộc) get_db.
  • models/: Khai báo các mô hình SQLAlchemy ORM - ánh xạ quan hệ đối tượng SQLAlchemy đại diện cho các bảng cơ sở dữ liệu vật lý cùng các chỉ mục, ràng buộc và mối quan hệ của chúng.
  • schemas/: Định nghĩa các mô hình Pydantic để xác thực payload (tải trọng dữ liệu) của HTTP request - yêu cầu HTTP đầu vào (ví dụ: BookCreate) và làm sạch response body - thân yêu cầu phản hồi đầu ra (ví dụ: BookResponse).
  • api/: Tách biệt các endpoint (điểm cuối) một cách logic thành các bộ định tuyến phụ (ví dụ: books.py, authors.py) bằng cách sử dụng APIRouter của FastAPI, sau đó chúng được hợp nhất vào ứng dụng chính.

5. End-to-End Request-Response Pipeline - Đường ống Yêu cầu-Phản hồi Toàn diện

Hiểu được request-response pipeline - đường ống yêu cầu-phản hồi (hay request/response lifecycle - chu kỳ sống yêu cầu/phản hồi) là rất quan trọng để theo dõi quá trình thực thi và gỡ lỗi. Biểu đồ dưới đây mô tả chi tiết trình tự của một request (yêu cầu) tải lên tệp tin dạng multipart nhắm mục tiêu đến endpoint (điểm cuối) tải lên hình ảnh bìa.

Detailed Step-by-Step Walkthrough - Hướng dẫn chi tiết Từng Bước

Phase 1: Bootstrapping & Mounting - Khởi động & Gắn kết

Khi ứng dụng khởi động, Uvicorn khởi tạo đối tượng FastAPI được định nghĩa trong app/main.py. Trong quá trình khởi động này, các routes (tuyến đường) được đăng ký và việc mounting (gắn kết) thư mục tệp tĩnh diễn ra:

from fastapi import FastAPI
from fastapi.staticfiles import StaticFiles
from app.api import books, authors, categories

app = FastAPI(title="Book Management API")

# Mount the static directory to serve book covers directly
app.mount("/static", StaticFiles(directory="app/static"), name="static")

# Include the routers for modular API routing
app.include_router(books.router, prefix="/api/books", tags=["Books"])
app.include_router(authors.router, prefix="/api/authors", tags=["Authors"])
app.include_router(categories.router, prefix="/api/categories", tags=["Categories"])

Phase 2: Routing & Parsing - Định tuyến & Phân tích cú pháp

Khi nhận được một HTTP request - yêu cầu HTTP đến (ví dụ: POST /api/books/42/cover), bộ định tuyến nội bộ của FastAPI sẽ khớp phương thức request (yêu cầu) và đường dẫn với hàm hoạt động đường dẫn. Các path parameter - tham số đường dẫn (chẳng hạn như 42) được trích xuất từ URL và được phân tích cú pháp thành các kiểu dữ liệu Python đã khai báo của chúng.

Phase 3: Dependency Resolution (get_db Lifecycle) - Giải quyết Phụ thuộc (Vòng đời của get_db)

Trước khi thực thi route handler - bộ xử lý định tuyến, FastAPI sẽ giải quyết bất kỳ dependency (phụ thuộc) nào được khai báo. Phụ thuộc get_db được giải quyết thông qua một generator:

  1. FastAPI gọi hàm phụ thuộc get_db().
  2. SessionLocal() được gọi để tạo một kết nối cơ sở dữ liệu phiên SQLAlchemy mới.
  3. Hàm thực thi cho đến câu lệnh yield db, cung cấp phiên hoạt động cho hàm đường dẫn.
  4. Hàm hoạt động đường dẫn chạy.
  5. Sau khi response (phản hồi) được lập ra, generator tiếp tục hoạt động, thực thi khối finally gọi lệnh db.close(). Điều này đảm bảo các kết nối cơ sở dữ liệu được trả lại cho pool (nhóm kết nối), ngăn ngừa rò rỉ bộ nhớ.

Phase 4: Route Handlers (Detailed function analysis) - Route Handler - bộ xử lý định tuyến (Phân tích chi tiết chức năng)

1. Retrieve Book List (list_books) - Truy xuất Danh sách Sách (list_books)

Endpoint (điểm cuối) này xử lý việc truy xuất danh sách và triển khai lọc cũng như phân trang bằng cách sử dụng các query parameter - tham số truy vấn tùy chọn. Truy vấn SQL được xây dựng động bằng cách sử dụng các bộ lọc có điều kiện:

@router.get("/", response_model=List[schemas.Book])
def list_books(
    skip: int = 0,
    limit: int = 100,
    author_id: Optional[int] = None,
    category_id: Optional[int] = None,
    keyword: Optional[str] = None,
    db: Session = Depends(get_db)
):
    """
    Retrieves a list of books with pagination and dynamic filtering.
    """
    # Initialize base query
    query = db.query(models.Book)
    
    # Conditionally apply filters
    if author_id:
        query = query.filter(models.Book.author_id == author_id)
    if category_id:
        query = query.filter(models.Book.category_id == category_id)
    if keyword:
        # Case-insensitive search inside title or description
        query = query.filter(
            (models.Book.title.ilike(f"%{keyword}%")) | 
            (models.Book.description.ilike(f"%{keyword}%"))
        )
    
    # Execute paginated query
    return query.offset(skip).limit(limit).all()
2. Create Book Record (create_book) - Tạo Bản ghi Sách (create_book)

Trước khi thêm sách, chúng tôi thực hiện các kiểm tra xác thực để đảm bảo tính toàn vẹn của cơ sở dữ liệu. Chúng tôi kiểm tra xem Tác giả và Danh mục được tham chiếu có tồn tại trước khi chèn hay không:

@router.post("/", response_model=schemas.Book, status_code=status.HTTP_201_CREATED)
def create_book(book_in: schemas.BookCreate, db: Session = Depends(get_db)):
    """
    Creates a new book after verifying parent entities exist.
    """
    # 1. Verify author exists
    author_exists = db.query(models.Author).filter(models.Author.id == book_in.author_id).first()
    if not author_exists:
        raise HTTPException(status_code=404, detail="Author not found")
        
    # 2. Verify category exists
    category_exists = db.query(models.Category).filter(models.Category.id == book_in.category_id).first()
    if not category_exists:
        raise HTTPException(status_code=404, detail="Category not found")
        
    # 3. Save book record
    db_book = models.Book(**book_in.model_dump())
    db.add(db_book)
    db.commit()
    db.refresh(db_book)
    return db_book
3. Update & Delete Book Records - Cập nhật & Xóa các Bản ghi Sách

Cập nhật các bản ghi cơ sở dữ liệu yêu cầu tải thực thể, áp dụng các thuộc tính mới và thực hiện database transaction - giao dịch cơ sở dữ liệu:

@router.put("/{book_id}", response_model=schemas.Book)
def update_book(book_id: int, book_in: schemas.BookUpdate, db: Session = Depends(get_db)):
    """
    Modifies an existing book record. Only updates modified fields.
    """
    db_book = db.query(models.Book).filter(models.Book.id == book_id).first()
    if not db_book:
        raise HTTPException(status_code=404, detail="Book not found")
        
    # Extract only provided update parameters
    update_data = book_in.model_dump(exclude_unset=True)
    for key, value in update_data.items():
        setattr(db_book, key, value)
        
    db.commit()
    db.refresh(db_book)
    return db_book
 
@router.delete("/{book_id}", status_code=status.HTTP_204_NO_CONTENT)
def delete_book(book_id: int, db: Session = Depends(get_db)):
    """
    Removes a book from the inventory.
    """
    db_book = db.query(models.Book).filter(models.Book.id == book_id).first()
    if not db_book:
        raise HTTPException(status_code=404, detail="Book not found")
        
    db.delete(db_book)
    db.commit()
    return None
4. Upload Book Cover Image (upload_book_cover) - Tải lên Hình ảnh Bìa Sách (upload_book_cover)

Endpoint (điểm cuối) này nhận một payload (dữ liệu tải) nhị phân thông qua multipart/form-data. Nó thực hiện các kiểm tra xác thực bảo mật (kích thước và định dạng tệp), lưu tệp cục bộ bằng các thao tác với tệp của Python, và cập nhật cơ sở dữ liệu với liên kết tĩnh:

import os
import uuid
from fastapi import APIRouter, Depends, HTTPException, UploadFile, File, status
from sqlalchemy.orm import Session
 
router = APIRouter()
 
# Security constants
MAX_FILE_SIZE = 2 * 1024 * 1024  # 2MB Limit
ALLOWED_MIME_TYPES = ["image/jpeg", "image/png"]
ALLOWED_EXTENSIONS = [".jpg", ".jpeg", ".png"]
 
@router.post("/{book_id}/cover", response_model=schemas.Book)
async def upload_book_cover(
    book_id: int,
    file: UploadFile = File(...),
    db: Session = Depends(get_db)
):
    # 1. Verify book exists
    db_book = db.query(models.Book).filter(models.Book.id == book_id).first()
    if not db_book:
        raise HTTPException(status_code=404, detail="Book not found")
        
    # 2. Validate file extension
    _, ext = os.path.splitext(file.filename.lower())
    if ext not in ALLOWED_EXTENSIONS:
        raise HTTPException(
            status_code=status.HTTP_400_BAD_REQUEST,
            detail=f"Invalid file extension. Allowed extensions: {', '.join(ALLOWED_EXTENSIONS)}"
        )
        
    # 3. Validate MIME type
    if file.content_type not in ALLOWED_MIME_TYPES:
        raise HTTPException(
            status_code=status.HTTP_400_BAD_REQUEST,
            detail=f"Invalid MIME type. Allowed types: {', '.join(ALLOWED_MIME_TYPES)}"
        )
        
    # 4. Validate file size (Read up to 2MB to verify size)
    file.file.seek(0, os.SEEK_END)
    file_size = file.file.tell()
    file.file.seek(0)  # Reset pointer back to beginning
    
    if file_size > MAX_FILE_SIZE:
        raise HTTPException(
            status_code=status.HTTP_400_BAD_REQUEST,
            detail="File size exceeds the maximum allowed limit of 2MB."
        )
        
    # 5. Generate secure unique filename to avoid collision or path traversal
    unique_filename = f"{uuid.uuid4()}{ext}"
    upload_dir = "app/static/covers"
    os.makedirs(upload_dir, exist_ok=True)
    file_path = os.path.join(upload_dir, unique_filename)
    
    # 6. Save binary stream to file system asynchronously
    try:
        with open(file_path, "wb") as buffer:
            while chunk := await file.read(1024 * 64):
                buffer.write(chunk)
    except Exception as e:
        raise HTTPException(
            status_code=status.HTTP_500_INTERNAL_SERVER_ERROR,
            detail="Could not write file content to disk."
        )
        
    # 7. Persist static URL to the database
    static_url = f"/static/covers/{unique_filename}"
    db_book.cover_image = static_url
    db.commit()
    db.refresh(db_book)
    
    return db_book

Phase 5: Cleanup & Serialization - Dọn dẹp & Tuần tự hóa dữ liệu

Sau khi route handler - bộ xử lý định tuyến trả về kết quả, FastAPI xử lý serialization (tuần tự hóa dữ liệu) và session teardown - kết thúc phiên:

  • Serialization - Tuần tự hóa dữ liệu: Thực thể cơ sở dữ liệu được trả về bởi route handler - bộ xử lý định tuyến được phân tích cú pháp bởi Pydantic schema - sơ đồ Pydantic khai báo trong response_model. Pydantic lọc bỏ các thuộc tính nội bộ và nhạy cảm (ví dụ: mật khẩu hoặc các đường dẫn tệp cục bộ) và ánh xạ các bản ghi DB bằng các thuộc tính.
  • Connection release - Giải phóng kết nối: FastAPI gọi bước dọn dẹp thực thi của bộ giải quyết phụ thuộc. Trong trường hợp của get_db, điều này kích hoạt khối finally, thực thi db.close() để dọn dẹp các kết nối cơ sở dữ liệu một cách an toàn.

6. Advanced Topics & Production Best Practices - Các Chủ đề Nâng cao & Thực tiễn Tốt nhất trong Môi trường Vận hành

Khi ứng dụng mở rộng quy mô từ môi trường phát triển sang production environments - môi trường vận hành thực tế, các khả năng nâng cao như cross-origin resource sharing - chia sẻ tài nguyên nguồn gốc chéo (CORS), middleware (phần mềm trung gian) tùy chỉnh, websocket (giao thức websocket) hai chiều thời gian thực và cấu hình triển khai được tối ưu hóa trở nên quan trọng để xây dựng các API mạnh mẽ, an toàn và hiệu năng cao.

Cross-Origin Resource Sharing (CORS) - Chia sẻ Tài nguyên Nguồn gốc Chéo

CORS là một cơ chế bảo mật do trình duyệt thực thi nhằm ngăn chặn các ứng dụng web chạy trên một miền thực hiện các yêu cầu tài nguyên đến một miền khác, trừ khi được server (máy chủ) cho phép một cách rõ ràng. Để cho phép các frontend framework - khung công việc phía máy khách (như React, Vue hoặc Svelte) tương tác với backend (phía máy chủ) FastAPI của bạn, bạn phải cấu hình CORSMiddleware.

from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware

app = FastAPI()

# Define list of origins allowed to make requests to this API
origins = [
    "http://localhost:3000",      # Common React dev server port
    "https://myfrontend.com",     # Production frontend domain
]

app.add_middleware(
    CORSMiddleware,
    allow_origins=origins,            # Allows requests from specific origins
    allow_credentials=True,           # Allows cookies or authentication headers
    allow_methods=["*"],              # Allows all HTTP methods (GET, POST, etc.)
    allow_headers=["*"],              # Allows all request headers
)

Custom Middleware: Execution Time Tracker - Middleware Tùy chỉnh: Bộ theo dõi Thời gian Thực thi

Middleware (phần mềm trung gian) là một hàm chạy trước khi mỗi request (yêu cầu) được xử lý bởi một route handler - bộ xử lý định tuyến cụ thể, và cũng chạy sau khi mỗi response (phản hồi) được tạo ra. Các middleware (phần mềm trung gian) là lý tưởng cho các mối quan tâm chung xuyên suốt như logging (ghi nhật ký), xác thực yêu cầu và profiling (phân tích hiệu năng).

import time
from fastapi import Request, FastAPI

app = FastAPI()

@app.middleware("http")
async def add_process_time_header(request: Request, call_next):
    start_time = time.perf_counter()
    
    # Process the request and pass it to the route handler
    response = await call_next(request)
    
    # Calculate execution duration
    process_time = time.perf_counter() - start_time
    
    # Inject execution duration in response headers
    response.headers["X-Process-Time"] = f"{process_time:.4f}s"
    
    return response

Real-Time Communication with WebSockets - Giao tiếp Thời gian thực với WebSockets (Giao thức WebSocket)

WebSockets (giao thức websocket) cho phép các kênh giao tiếp hai chiều, song công và lâu dài trên một kết nối TCP duy nhất, điều này lý tưởng cho các ứng dụng thời gian thực như hệ thống trò chuyện, bảng điều khiển trực tiếp và hệ thống thông báo.

from fastapi import FastAPI, WebSocket, WebSocketDisconnect

app = FastAPI()

class ConnectionManager:
    def __init__(self):
        self.active_connections: list[WebSocket] = []

    async def connect(self, websocket: WebSocket):
        await websocket.accept()
        self.active_connections.append(websocket)

    def disconnect(self, websocket: WebSocket):
        self.active_connections.remove(websocket)

    async def send_personal_message(self, message: str, websocket: WebSocket):
        await websocket.send_text(message)

    async def broadcast(self, message: str):
        for connection in self.active_connections:
            await connection.send_text(message)

manager = ConnectionManager()

@app.websocket("/ws/{client_id}")
async def websocket_endpoint(websocket: WebSocket, client_id: int):
    await manager.connect(websocket)
    try:
        while True:
            # Wait for any incoming client text message
            data = await websocket.receive_text()
            await manager.send_personal_message(f"You wrote: {data}", websocket)
            await manager.broadcast(f"Client #{client_id} says: {data}")
    except WebSocketDisconnect:
        manager.disconnect(websocket)
        await manager.broadcast(f"Client #{client_id} has left the chat")

Production Deployment - Triển khai trên Môi trường Vận hành Thực tế

Đối với production environments - môi trường vận hành thực tế, các ứng dụng FastAPI thường được chạy phía sau một ASGI server - máy chủ ASGI như Uvicorn hoặc Gunicorn (quản lý các uvicorn workers - tiến trình làm việc uvicorn) kết hợp với một reverse proxy - ủy quyền ngược như Nginx để chấm dứt kết nối SSL, cân bằng tải và phục vụ tệp tĩnh.

Uvicorn Command Settings - Cài đặt Lệnh Uvicorn

Chạy Uvicorn với nhiều workers (tiến trình làm việc) trong môi trường vận hành thực tế để tận dụng các CPU đa lõi. Một quy tắc chung phổ biến là (2 * số_lượng_lõi_CPU) + 1 workers.

# Run FastAPI in production with 4 worker processes, binding to all interfaces

Nginx Reverse Proxy Configuration Block - Khối Cấu hình Nginx Reverse Proxy - Ủy quyền Ngược Nginx

Nginx chặn lưu lượng HTTP/HTTPS đến, xử lý các client (máy khách) chậm, phục vụ trực tiếp các tài sản tĩnh và ủy quyền các yêu cầu ứng dụng đến ASGI server - máy chủ ASGI backend (phía máy chủ).

server {
    listen 80;
    server_name api.example.com;

    # Redirect all HTTP traffic to HTTPS (Recommended for Production)
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name api.example.com;

    ssl_certificate /etc/letsencrypt/live/api.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/api.example.com/privkey.pem;

    client_max_body_size 10M; # Max file upload size

    # Route request to the FastAPI application
    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # Support WebSockets protocols
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }

    # Direct static asset caching
    location /static/ {
        alias /var/www/fastapi_app/app/static/;
        expires 30d;
        add_header Cache-Control "public, no-transform";
    }
}
